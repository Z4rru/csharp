Understanding Interfaces

The foundation of flexible, maintainable code

What is an Interface? (In Plain English)

üîå

Real-World Analogy: Think of a power outlet.

Every device that plugs into an outlet must have the right plug shape (the interface). The outlet doesn't care if you plug in a lamp, TV, or phone charger ‚Äì it just needs the right shape plug.

Similarly, an interface defines what methods a class must have, but not how those methods work.

Key Rules to Remember:

  • ‚úì Interfaces cannot contain implementation code ‚Äì only signatures (names and parameters)
  • ‚úì Interface names start with capital "I" (e.g., ITransactions, IShape)
  • ‚úì Classes that implement an interface must provide all methods defined in the interface
  • ‚úì A class can implement multiple interfaces (unlike inheritance, which allows only one parent class)
